# CKEditor 4 Configuration for WeChat Article Compatibility

To achieve 1:1 fidelity with WeChat articles and prevent the editor from stripping or modifying semantic tags (like `<section>`) and styles, please apply the following configuration to your CKEditor 4 instance.

## 1. Disable Advanced Content Filter (ACF)

The most important step is to disable the filter that strips "unknown" tags or styles.

Open your `config.js` file and add:

```javascript
CKEDITOR.editorConfig = function( config ) {
    // ... existing config ...

    // ALLOW ALL CONTENT
    // This disables the filter and allows any HTML tag, class, or style.
    config.allowedContent = true;

    // Optional: If you want to be more specific instead of allowing everything:
    // config.extraAllowedContent = 'section(*); div(*); style; script; iframe; p(*); span(*)';

    // DISABLE CONTENT FILTERING
    // Prevent the editor from "correcting" HTML
    config.fillEmptyBlocks = false;
    config.basicEntities = false;
    config.autoParagraph = false; // Prevent wrapping content in <p> tags automatically
};
```

## 2. Preserve Semantic Tags

WeChat articles heavily use `<section>` tags for layout. Ensure these are not stripped. If `allowedContent = true` is set, this should happen automatically.

## 3. Enable Source Editing

Ensure your editor toolbar has the 'Source' button so you can paste the HTML generated by this tool directly.

```javascript
config.toolbar = [
    { name: 'document', items: [ 'Source', '-', 'Save', 'NewPage', 'Preview', 'Print', '-', 'Templates' ] },
    // ... other toolbar items
];
```

## 4. Paste Configuration

When pasting content, ensure the editor doesn't try to "clean" it aggressively.

```javascript
config.pasteFilter = null; // Disable paste filtering
```
